Sprint Completed Stories

1. Bootstrap

  • Create placeholder pages: /, /about, /events/new, /events/0, /events/1, /events/2
  • Apply Bootstrap CSS framework to every page
  • Apply header and footer elements to each page
  • Place a link to the about page and the homepage in the footer element of every page

2. Homepage

  • Create and display team logo
  • Create unordered list of events, where each event is in an li element with class="event" and id="event-{id}", where {id} indicates the numerical id of the event.
  • Make sure each list element displays pertinent information about the event, including: title, date, and how many people are RSVPd etc.
  • Make sure every event li element has an HTML5 time tag that has a datetime attribute equal to the time in ISO 8601 format.
  • Make sure that the title for each event is a clickable link to the "detail page" for that event, which should be at /events/{{id}}, where id refers to the id of the event, e.g. /events/0 would take me to the event with id zero.
  • Make sure events that are over are not shown
  • Make sure users can click a link to take them to /events/new. The link has id="new"

We started by getting all of our teammates logged into and connected across the various platforms. We then got to the business of converting the user stories provided on the “Project” site into a more usable format. After Chris did some only-kind-of-innovative work by putting the data into a Google doc., Cliff and Bridgette did some actually-innovative work by putting the data into a Google spreadsheet. This allowed the team to discuss and document our perceptions of how difficult work would be, so we could assign story points.

Once we assigned story points, we decided to split them between the three sprints, with about half as many in the first sprint as in the second two sprints. We thought this would be reasonable, as we ramp up our skills early on and things take longer.
